Commonly Used Graphic File Formats

Graphic types commonly used on web pages include: 

  • GIF
  • JPG
  • PNG
  • WebP
  • SVG

    GIF

    GIF stands for Graphics Interchange Format. The animated GIF has made and kept this format popular, but its format can be used for still graphics as well. Some GIF facts: 

    • Best used for line art and logos
    • Maximum of 256 colors
    • One color can be configured as transparent
    • Can be animated
    • Uses lossless compression (data compression that reduces file sizes without sacrificing any significant information in the process)
    • Can be interlaced (Displays low-resolution versions of the image in a browser as the file downloads. Interlacing makes download time seem shorter but also increases the file size.)

      JPG

      JPG stands for Joint Photographic Experts Group and is one of the most common formats for photographs. Some JPG facts include: 

      • Best used for photographs
      • Up to 16.7 million colors
      • Use lossy compression (used when a file can afford to lose some data or if storage space needs to be considered.)
      • Cannot be animated
      • Cannot be made transparent

      PNG

      PNG stands for Portable Network Graphic. Some PNG facts include: 

      • Supports millions of colors
      • Supports multiple levels of transparency 
      • Supports interlacing
      • Uses lossless compression
      • Combines the best of GIF & JPEG
      • Well-supported by modern browsers

      WebP

      WebP is a modern image format that provides superior lossless and lossy compression for images on the web. Using WebP, webmasters and web developers can create smaller, richer images that make the web faster. Learn more here! opens in new window

      WebP Facts include: 

      • Supports millions of colors
      • Supports transparency
      • Uses both lossy and lossless compression
      • Intended to provide improved compression for GIF and JPG images
      • Browser support is growing

      SVG

      An SVG file, short for scalable vector graphic file, is a standard graphics file type used for rendering two-dimensional images online. SVGs are a bit of a magic trick in web development. They generate clean, crisp graphics at any scale and are optimized for search engines. SVG files are really images at all. They do not contain pixels of images. They are programmable and capable of dynamic animations. SVG Facts include: 

      • Vector image format, ideal for user interface elements, icons, diagrams, etc.,
      • can be drawn accurately at different sizes.
      • Tiny!

      Adding Images HTML Pages

      Remember, HTML Is just a text file that contains marked-up content. It can't contain an image file like PowerPoint or Word document. It can, however, include an image via the image tag. 

      <img src="dog.jpg" alt="Dog at computer" height="100" width="100">

      src  = File name AND LOCATION of the graphic
      alt = Configures alternate text description - Required for Accessibility. Use "decorative" if the graphic is not part of your content.
      height- Height of the graphic in pixels
      width - Width of the graphic in pixels 

      Graphic Considerations

      Graphics should always be optimized. What is the smaller file size I can use without losing quality? Dimensions should be reduced of the image to the actual width and height of the image on the web page.

      Image Naming Best Practices

      • Use all lowercase letters 
      • Do not use punctuation symbols and spaces
      • Do not change the file extensions  (should be .gif, .jpg, .jpeg, or .png)
      • Keep your file names short but descriptive
        • db.gif is probably too short
        • myimagewithmydogonmybirthday.gif is too long
        • dogbday.gif may be just about right

      HTML or CSS?

      This notes page has focused on the different types of graphics and how to embed them using HTML. However, based on your readings for this module, you also learned how to include images on a web page via CSS. 

      The general rule of thumb is simple on this: 

      • If the image is part of the content, use the HTML img tag with all accessibility attributes.
      • If the image serves an aesthetic purpose, like adding visual flare, a texture, etc, and is not part of the content, then that image should be included via CSS and not HTML. 

      Remember, HTML is for your content. CSS is for layout and design. If you remove the CSS, the content should still be accessible and contain any images the content requires.
